# Критерии оценивания проекта

## Структура проекта

### Название задания/проекта

- название проекта в README лаконично (состоит из 1-3 слов), отражает смысловую нагрузку проекта
- название проекта в REAMDE выделено как заголовок первого уровня в Markdown (#)

### Краткая аннотация

- Резюме содержит: о чем проект, что он включает в себя с точки зрения знаний/навыков
- Расположен после названия проекта

### Содержание проекта

Оглавление проекта выглядит так: Глава 1, Глава 2, Глава 3 и т.д.

### Разделы (Chapter/Part) 

Раздел (Chapter) включает:
- название (заголовок), отражающее смысловое содержание данного раздела; пишется с новой строки
- теоретический и/или практический (в виде задач/упражнений) материал

### Задачи/упражнения (Task/Exercise)

Задача/упражнение содержит: 
- название, раскрывающее суть дальнейших практических действий
- описание, отражающее конкретную цель; включает определенные шаги (этапы) выполнения практических действий

## Смысловая нагрузка материала

### Соответствие заявленной теме

Представленный материал в полной мере раскрывает заявленную тему проекта 

### Полнота информации

- Наполнение материала задач/проекта соответствует заявленным образовательным результатам
- Подсказки и описание проекта позволяют участнику корректно найти информацию и выполнить проект

### Образовательный результат

В тексте задачи понятно отражен конечный результат, который надо получить 

В задаче прописано в каком подходящем формате нужно загрузить решение

- для коммерции: в файлообменик 
- для основы: в репозиторий

### Понятность и доступность

Если смысловая трактовка задачи неоднозначна, допускается пояснение специализированных терминов доступным языком

### Самостоятельность задач

Задачи внутри проекта целостны, самостоятельны

### Единый сторителлинг

Прослеживается единая история, т.е. все задачи выстраиваются в единый проект, разделенный на конкретные задачи

## Практическая значимость 

### Практикоориентированность

Содержание проекта направлено на практические действия студента

### Оценивание Р2Р проверкой

Заявленный результат задачи (проекта) возможно проверить Р2Р проверкой

## Стилистика и формат материала

### Логичность текста

Материал излагается последовательно, противоречия отсутствуют, структура предложений логична

### Единый стиль и язык текста 

Материал излагается в едином стиле и с одной линией повествования. 

Материал соответствует:
- Tone of Voice (обращение к пирам на «ты»)
- обладает единой терминологией, общностью, целостностью и объемом

### Грамотность текста

Отсутствие синтаксических, грамматических и орфографических ошибок в тексте

### Соответствие форматов содержанию

Формат подачи информации соответствует заявленной теме и предметному направлению, а также формату передачи информации

### Разнообразие форматов

Текст проекта интуитивно свободно воспринимается и включает разные форматы, при необходмости:
- таблицы
- ссылки 
- схемы
- иллюстрации
- видео (добавлено через медиатеку)
- PDF, аудио (добавлено через файловый сервис)

### Корректность указанного формата

- Указанный формат корректно оформлен и открывается 
- Содержащая информация релевантна и актуальна

### Дополнительные материалы

Дополнительные материалы загружены: 
- в файлообменник
- в директорию materials 

"Дополнительные источники предоставлены в безопасном формате, без потери информации и доступа по истечению времени. 

Например: по возможности статьи даны не формате ссылок, а отдельными файлами
