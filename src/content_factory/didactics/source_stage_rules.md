# Source stage rules from `исходник`

Этот файл фиксирует runtime-правила, извлеченные из PDF в папке `исходник`.
Он нужен как короткий контракт для агентов, чтобы публичный README совпадал
с didactics-артефактами и исходным поэтапным промптом.

## Публичная структура README

- H1: лаконичное название проекта из 1-3 слов.
- Под H1 идет короткая аннотация без пересказа глав.
- Далее идет `## Содержание`.
- Обязательные главы:
  - `## Глава 1. Введение и инструкция`
  - `## Глава 2. Теоретический блок`
  - `## Глава 3. Практический блок`
- Теория оформляется заголовками `### 2.N. Название`.
- Практика оформляется заголовками `### Задание N. Название`.
- Бонусный раздел опционален.
- Финальный раздел завершает текущий проект и не анонсирует следующий проект.

## Шаблон практического задания

Каждое публичное задание в README должно содержать эти блоки в указанном порядке:

1. `**Что нужно сделать**`
2. `**Что должно получиться**`
3. `**Ограничения и условия**` при наличии ограничений, рисков или рамок
4. `**Формат сдачи**`
5. `**Переход к следующему заданию**`

Внутренние поля генератора могут храниться как situation/input/goal/approach/result,
но в публичном README они должны быть сведены к этому шаблону.

## P2P-проверяемость

- Результат задания должен быть наблюдаемым.
- В блоке `Что должно получиться` нужно давать 2-5 проверяемых признаков.
- Если результат лежит в репозитории или файле, путь должен быть указан явно.
- Задания должны образовывать цепочку: следующий шаг использует артефакт предыдущего,
  когда это соответствует проекту.

## Материалы и самостоятельность

- Materials являются raw evidence, а не готовыми решениями.
- Задание не должно выдавать студенту финальный ответ или заполненный артефакт.
- Формат сдачи описывает, что показать на p2p-ревью, а не как получить готовое решение.

## Проверка инструментов

- Официальные ссылки добавляются только для инструментов, которые явно есть в учебном плане.
- Если инструмент неизвестен для локального mapping, генератор не выдумывает ссылку и должен оставить warning для review.
