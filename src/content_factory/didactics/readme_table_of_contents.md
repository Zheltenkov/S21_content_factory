

## Основные положения формирования содержания контента

1. Содержание (Table of Contents) должно быть структурированным по основным частям структуры проекта
2. Структурирование происходит по средствам маркированного списка (`- Text`)

Формат структуры:

```md
- Пункт 1
  - Подпункт 1.1
    - Подпункт 1.1.1
```

3. Каждый пункт содержания проекта должен иметь относительную ссылку на заголовок markdown

Например:

```md
- [Глава 1](#глава-1)
```

4. В содеражание не должны входить:
- Название проекта (заголовок первого уровня)
- Заголовок содержания проекта (заголовок, который обозначает блок с содержанием проекта)
- Вторичные блоки контента проекта

5. Заголовок блока с содержанием должен строго быть второго уровня (`##`) и его название должно быть таким:

- На русском языке: `## Содержание`

## Шаблонный пример формирования содержания контента

Пример формирования содержания проекта к шаблонному проекта (шаблонный проект представлен в <template_rules>):

```md
## Содержание

 - [Глава 1. Введение и инструкция](#глава-1-введение-и-инструкция)
   - [Введение](#введение)
   - [Инструкция](#инструкция)
 - [Глава 2. Теоретический блок](#глава-2-теоретический-блок)
   - [2.1. \<название пункта теоретического блока 1\>](#21-название-пункта-теоретического-блока-1)
   - [2.2. \<название пункта теоретического блока 2\>](#22-название-пункта-теоретического-блока-2)
   - [2.K. \<название пункта теоретического блока K\>](#23-название-пункта-теоретического-блока-K)
 - [Глава 3. Практический блок](#глава-3-практический-блок)
   - [Задание 1. \<название задания 1\>](#задание-1-название-задания-1)
   - [Задание 2. \<название задания 2\>](#задание-2-название-задания-2)
   - [Задание N. \<название задания N\>](#задание-2-название-задания-n)
   - [Бонусное задание 1\*: \<название бонусного задания 1\>](#бонусная-задача-1-название-бонусного-задания-1)
   - [Бонусное задание M\*: \<название бонусного задания M\>](#бонусная-задача-m-название-бонусного-задания-2)

```
