КРИТИЧЕСКИ ВАЖНО:
- Контент файла должен относиться ИМЕННО к этому проекту, а не к абстрактной бизнес-теме.
- Не подменяй домен проекта случайными темами вроде экологии, маркетинга, продаж или устойчивого развития, если их нет в контексте.
- Примеры ниже показывают только ФОРМАТ, а не тему, которую нужно копировать.
- Файл должен быть входным материалом для самостоятельной работы студента. Не включай готовое решение, итоговую классификацию, готовые рекомендации, финальную матрицу, стратегию, план или отчет, если студент должен создать это в задаче.
- Для аналитических задач генерируй сырые кейсы, инциденты, интервью, ограничения, наблюдения, требования, логи и фрагменты переписки, но не готовый итоговый артефакт.
- Если ожидаемый результат задачи — реестр, таблица, матрица, план, стратегия или сравнительный документ, входной файл должен выглядеть как рабочие заметки и разговоры: не нумеруй готовые итоговые элементы и не добавляй финальные поля, которые студент должен заполнить сам.

ТРЕБОВАНИЯ:
1. Сгенерируй реалистичные данные нужного объёма:
   - для csv/json/txt с табличной структурой — {rows_count} строк/записей, обычно 50-100
   - для md — 30-100 строк сырого evidence без готового решения
2. Строго следуй описанию структуры выше
3. Используй разнообразные, реалистичные значения
4. Данные должны быть валидными и готовыми к использованию

ФОРМАТ ОТВЕТА:

ЕСЛИ file_type = "csv":
Верни CSV содержимое в markdown блоке:
```csv
column1,column2,column3
value1,value2,value3
...
```

ЕСЛИ file_type = "json":
Верни JSON объект в формате:
{{
  "data": [
    {{"field1": "value1", "field2": "value2"}},
    ...
  ]
}}
Или просто массив, если это массив данных:
[
  {{"field1": "value1"}},
  ...
]

ЕСЛИ file_type = "txt":
Верни структурированный текст с разделителями (табуляция или |):
field1	field2	field3
value1	value2	value3
...

ЕСЛИ file_type = "md" (Markdown - для писем, чатов, шаблонов, примеров):
Верни текстовый контент в формате Markdown.
В начале MD-файла добавь блок:
## EvidenceSpec
- Тип evidence: <кейсы/интервью/переписка/требования/логи/заметки/наблюдения>
- Содержит: <какие сырые факты и сигналы есть в файле>
- Не содержит: <какие выводы, классификации, решения или итоговые поля студент должен вывести сам>
- Студент должен вывести: <2-4 пункта самостоятельной работы>

ВАЖНО ДЛЯ MD ФАЙЛОВ:
- Генерируй РЕАЛИСТИЧНЫЙ контент с именами, датами, контекстом
- Добавляй НАМЕРЕННЫЕ ОШИБКИ если это требуется для анализа (клише, неясные формулировки, отсутствие CTA)
- Используй форматирование Markdown (заголовки, цитаты, списки)
- Объём: 30-100 строк в зависимости от типа контента
- Не включай готовый итоговый артефакт, финальную классификацию, заполненную матрицу, рекомендации или план, если студент должен создать это сам

ВАЖНО:
- НЕ генерируй меньше 50 строк для табличных данных (csv, json)
- НЕ генерируй больше 100 строк
- Все данные должны быть реалистичными и разнообразными
- Следуй описанию структуры максимально точно
- Для MD файлов: создавай осмысленный текстовый контент с контекстом
