# Критерии проверки README

Проверка README нужна не для блокировки генерации, а для управляемой доработки. Результат показывается методологу как таблица критериев: пройдено, предупреждения, не пройдено.

## Общая модель

- Всего используется 39 критериев.
- Каждый критерий возвращает статус, комментарий и технические детали для отладки.
- Hard-критерии влияют на итоговую оценку.
- Soft/editorial/semantic-критерии могут отображаться как предупреждения, если нарушение не должно откатывать результат.
- Отсутствие необязательного контекста не должно превращаться в fail.

## Группы критериев

| Группа | Что проверяет |
|---|---|
| Структура документа | Заголовок, содержание, главы, заключение, Markdown-ссылки. |
| Глава 1 | Введение, инструкция, понятность входа в проект. |
| Глава 2 | Теоретический блок, объем разделов, определения, примеры, связь с проектом. |
| Глава 3 | Практические задания, ожидаемые результаты, артефакты, p2p-проверяемость. |
| Сторителлинг | Единый нарративный фокус и связность между главами. |
| Финальное завершение | Заключение завершает текущий проект без анонса следующего. |

## Образовательные результаты

Отдельное обязательное поле с знаниями, умениями и навыками больше не используется. Если в контексте есть образовательные результаты, критерий 2.4.5 использует их как дополнительный сигнал. Если образовательные результаты не переданы, критерий не должен снижать итоговую оценку; максимум — предупреждение о том, что дополнительный контекст отсутствует.

## Как читать статусы

| Статус | Значение |
|---|---|
| Пройден | Критерий выполнен. |
| Предупреждение | Есть риск качества, но это не повод откатывать генерацию автоматически. |
| Не пройден | Нарушение мешает принять README без правки. |

## Как использовать в перегенерации

Непройденные критерии можно передать в запрос правки, но запрос должен оставаться содержательным:

```text
Исправь непройденные критерии по главе 2 без изменения структуры проекта.
Сохрани количество задач и не добавляй новые главы.
```

Если правка добавляет или удаляет главу, разрешены связанные изменения:

- содержание;
- номера заголовков;
- Markdown-ссылки;
- rubric/metrics после пересчета.

```powershell
python -m pytest tests/validators/rubric -q
python -m pytest tests/utils/test_static_ui_contracts.py -q
```
