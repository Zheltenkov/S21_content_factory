# Инструкция для методолога

Эта инструкция описывает работу в интерфейсе без внутренних технических деталей.

## Главное меню

В верхнем меню доступны вкладки:

- Главная — последние запуски и переход к инструментам.
- Генерация — создание учебного проекта.
- Проверка — проверка готового README.
- Перевод — перевод документов и видео.
- Инструкция — краткая справка по работе.

## Генерация проекта

1. Откройте вкладку “Генерация”.
2. Загрузите CSV с учебным планом.
3. Выберите проект из найденных строк.
4. Проверьте параметры проекта.
5. Выберите тип сторителлинга.
6. При необходимости включите методологический режим.
7. Запустите генерацию.

После завершения появятся вкладки результата:

- Итоговый README;
- Практика;
- Данные;
- Метрики;
- Отчет;
- Перегенерация.

Архив результата содержит `README.md`, `check-list.yml` и дополнительные файлы, если они были сформированы.

## Тип сторителлинга

Тип сторителлинга подсказывает генератору, как связывать задачи, теорию и контекст проекта.

Режим по умолчанию — SJM / моделирование среды. Он применяется прежде всего к практической части: ситуациям, задачам, артефактам и критериям. Теория поддерживает этот контекст, но не должна превращаться в отдельный рассказ.

Кратко по вариантам:

| Тип | Когда использовать |
|---|---|
| SJM / моделирование среды | Нужно показать путь пользователя, среды или рабочего процесса. |
| Рабочий кейс | Есть понятная рабочая ситуация и проблема, которую нужно решить. |
| Ролевая ситуация | Важно, кто принимает решения и какие роли участвуют. |
| Сценарий проекта | Нужно провести студента по этапам разработки проекта. |
| Сквозная история | Нужна единая повествовательная линия от начала до конца. |
| Без сторителлинга | Требуется нейтральный учебный материал без сценарной рамки. |

## Методологический режим

В методологическом режиме генератор останавливается на контрольных точках. В основном окне отображается результат этапа или сравнение изменений. Действия выполняются через чат методолога.

Основные кнопки:

- Продолжить — принять текущий этап и перейти дальше.
- Редактировать — написать правку.
- Принять — применить подготовленное изменение.
- Сравнить — посмотреть, что изменится.

Чтобы правка была точной, выберите фильтр области применения: текущий блок, название, аннотация, глава, критерии и так далее. Потом напишите обычным языком, что нужно изменить.

Пример:

```text
Упрости задание 2. Оставь структуру, но добавь более понятный ожидаемый результат и не меняй остальные задания.
```

## Перегенерация

Перегенерация нужна для точечных правок уже готового README.

1. Откройте вкладку “Перегенерация”.
2. Разверните “Части README для перегенерации”.
3. Выберите один или несколько разделов.
4. Для каждого раздела напишите, что исправить и что оставить.
5. Если правка общая, используйте поле “Общая правка”.
6. Запустите перегенерацию.
7. Откройте “Сравнить” и проверьте diff.

Если вы просите добавить или удалить главу, система должна обновить содержание, номера и ссылки. Это нормальное связанное изменение.

## Проверка README

1. Откройте вкладку “Проверка”.
2. Загрузите README.
3. Нажмите “Проверить README”.
4. Просмотрите вкладку “Критерии”.
5. Используйте фильтры: все, не пройдено, предупреждения, пройдено.

Предупреждение — это сигнал качества, а не обязательная ошибка. Непройденный критерий лучше исправить до передачи проекта дальше.

## Перевод

Документы:

1. Откройте “Перевод”.
2. Выберите режим “Документ”.
3. Загрузите файл.
4. Выберите язык.
5. Нажмите “Перевести”.
6. Скачайте результат после завершения.

Видео:

1. Переключитесь на “Видео”.
2. Загрузите видео.
3. Выберите язык.
4. Запустите перевод.
5. После завершения скачайте видео или субтитры во вкладке видео.

Для языков:

- таджикский — кириллица;
- кыргызский — кириллица;
- узбекский — латиница.

## Если что-то пошло не так

| Ситуация | Что делать |
|---|---|
| Запуск висит в активных задачах | Откройте запуск. Если он восстановим, продолжите или остановите. |
| При возврате к запуску появился 404 | Сервер временно не нашёл состояние запуска. Локальное состояние сохраняется: обновите статус или страницу и проверьте запуск ещё раз. |
| Появился 502, 503 или 504 | Это временная ошибка сервера, модели или долгого шага генерации. Подождите немного и повторите действие. |
| Ошибка токена, лимита или модели | Выбранная модель сейчас недоступна. Смените модель в переключателе ИИ или передайте ошибку администратору. |
| После правки текст не обновился | Нажмите “Сравнить”. Обычно сравнение появляется автоматически после сохранения правки. |
| Генератор повторяет случайную фразу из описания | Сузьте правку: укажите конкретный раздел, что исправить и что не трогать. |
| Перевод долго не завершается | Проверьте статус. Если ошибка повторяется, уменьшите файл или смените формат. |
| Диаграмма не отображается | Проверьте Markdown/Mermaid синтаксис или откройте увеличенный просмотр. |
| Не удается войти | Используйте аккаунт с доменом Школы 21. |
| README не проходит критерии | Используйте непройденные критерии как основу для правки. |
